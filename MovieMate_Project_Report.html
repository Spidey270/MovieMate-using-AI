INTRODUCTION

MovieMate is a web-based, AI-powered social movie recommendation platform designed to provide a structured, highly engaging digital space for cinephiles. The system aims to simplify movie discovery, social interaction, and content curation through an integrated and user-friendly interface. By combining essential social media functionalities with advanced Artificial Intelligence within a single platform, MovieMate enhances the movie-watching experience for users globally.

In many streaming environments, discovering the perfect movie can be challenging, especially when information and recommendations are generic or scattered across various sources. MovieMate addresses this issue by offering a centralized system where users can access a comprehensive movie catalog, save favorites to their personal wishlist, and manage their cinematic journey effectively. The platform enables users to interact socially through a Live Global Chat and Real-Time Direct Messaging via WebSockets.

A key feature of MovieMate is its integration with Gemini AI for dynamic, bespoke movie recommendations. Users can generate highly tailored suggestions based on their unique tastes without leaving the application. These background processes invoke seamless, interactive alert notifications so users are notified the moment their AI list is compiled or a friend sends a message.

Furthermore, the platform incorporates a dedicated Administrator Dashboard, enabling system administrators to monitor platform statistics, manage active users securely, and manually dispatch global or targeted broadcast notifications to the entire user base.

Overall, MovieMate serves as a comprehensive, technology-driven solution for modern social movie discovery. By integrating AI exploration, real-time communication, administrative validation, and instantaneous notification assistance, the system contributes to a highly organized, efficient, and user-centered entertainment platform.

EXISTING SYSTEM

Traditionally, the process of finding movies and interacting with fellow cinephiles is handled through fragmented platforms. Users must browse IMDb or Letterboxd for ratings, traverse to Reddit or Discord for chatting, and jump to streaming platforms hoping to find a good recommendation. Many existing solutions are fragmented or limited in functionality, making it difficult for users to efficiently organize their interests and maintain a connected social network in one place.

Key Characteristics of Existing Systems

  • Lack of a Unified Platform: Most current recommendation solutions focus only on storing reviews or only on forum tracking, but not both, leading to poor integration and inefficiency.
  • Manual Organization Methods: In many cases, users rely on separate apps, or multiple websites to manage wishlists and converse, resulting in disorganization.
  • Limited User Contribution Features: Existing platforms often provide restricted options for users to submit dynamic live messages directly to their friends interactively.
  • Absence of Generative AI Tracking: Users frequently lack a proper Large Language Model capable system natively hooked into the interface to synthesize vast movie databases.
  • Poor User Experience: Many existing applications have non-intuitive interfaces or lack visual modern responsiveness across devices, making navigation complex and less user-friendly.

Limitations of the Existing System

  • Inefficient Recipe Organization: The absence of a centralized digital system forces users to depend on scattered sources, making movie management time-consuming and unstructured.
  • Limited User Contribution: Separated communication domains discourage users from sharing and discussing their own opinions effectively.
  • No Structured Broadcast Monitoring: Users often lack real-time push tracking of platform maintenance, leading to poor planning and unexpected downtime.
  • Device Compatibility Issues: Many existing applications are not fully responsive or optimized for different devices, reducing accessibility and convenience.
  • Fragmented Management Approach: The lack of an integrated platform connecting discovery, chat networking, and administrative overview results in poor coordination and reduced user satisfaction.

PROPOSED SYSTEM

The proposed system, MovieMate, is a fully functional, web-based Social Recommendation System designed to address the limitations of existing manual forum management solutions. It enables users to browse, manage, and organize wishlists efficiently while also allowing registered users to submit live chat payloads directly to the global platform. Once registered, these accounts and metadata become accessible securely via JWT token verification protocols.

The system is developed using a modern and scalable technology stack — FastAPI & Python for backend core development, raw WebSockets for low-latency live streaming, React (JS/JSX) orchestrated by Vite with Tailwind CSS for the front-end interface, and a highly flexible MongoDB relational document database — ensuring a seamless, responsive, and user-friendly experience.

Key Features of the Proposed System

Multi-Device Responsive Design: MovieMate is developed with a responsive design approach utilizing Tailwind CSS, ensuring consistent performance across desktops, tablets, and smartphones. Users can access data and manage social activities anytime and from any device.

Dynamic and User-Centric Interface: The platform features a clean, glassmorphic dark-mode interface that simplifies movie browsing, friend management, and chat submission. Smooth navigation enhances overall usability.

User Module: Registered users can browse the digital movie library, manage their available friends list through a centralized dashboard, create watchlist nodes, and submit their own AI generation prompts. Users can also receive asynchronous real-time Bell Notifications when their queries map successfully.

Global Contributor Module: Users can submit new live messages to the worldwide platform, which are simultaneously tracked to local state management, encouraging massive community participation.

Admin Dashboard: Administrators manage the platform by reviewing database total aggregate statistics, monitoring user registration activities, terminating flagged accounts, and dispatching custom platform-wide System Broadcast Notifications to ensure smooth and secure operation.

ABOUT THE FRONT-END - REACT & TAILWIND

The front-end of the MovieMate system is developed using modern web technologies such as React JS (JSX logic) and Tailwind CSS. These technologies are responsible for designing and structuring the visual and interactive components of the web application. The front-end acts as the presentation layer of the system, allowing users to interact with the platform in a clear, organized, and remarkably fast Single Page Application (SPA) setting.

React JSX is used to create the structural foundation of the JavaScript components, mounting headings, interactive forms, button states, and navigational contexts dynamically without reloading the document. Tailwind CSS is used to enhance the visual appearance of the website by instantly applying inline utility classes for colors, fonts, spacing, alignment, constraints, and flexbox design. Tailwind also natively enables breakpoint responsive design, ensuring that the interface adapts flawlessly to different screen widths.

In the MovieMate system, the front-end is carefully orchestrated to provide an awe-inspiring user experience. Users can easily scroll horizontally through movie carousels, submit live socket connections, manage Wishlist persistence through context variables, and trigger backend jobs instantly. Well-structured Axios forms are implemented to collect user input efficiently, while clear navigation logic blocks non-authorized users using custom wrapper routes (e.g. ProtectedRoute, AdminRoute).

Responsive design principles are applied utilizing `lucide-react` iconography to ensure compatibility across desktops, tablets, and smartphones. This enhances accessibility and ensures that users can manage workflows conveniently from various devices. The front-end interface is engineered to minimize complexity, reduce DOM-paint errors, and provide hyper-smooth JSON interaction with backend processes.

Overall, the front-end of MovieMate plays a paramount role in delivering an engaging and highly performant user experience. It serves as the lightning-fast bridge between the viewer and the system’s backend logic, ensuring seamless WebSocket communication and impeccable visual delivery.

DATA FLOW DIAGRAM (DFD)

A Data Flow Diagram is a network that describes the flow of data and processes that change, or transform, data throughout the system. This network is constructed by use a set of symbols that do not imply a physical implementation. It is a graphical tool for structured analysis of the system requirements. DFD models a system by using external entities from which data flows to a process, which transforms the data and creates output-data-flows which go to other processes or external entities or files. Data in files may also flow to processes as inputs.

There are various symbols used in a DFD. Bubbles represent the processes. Named arrows indicate the data flow. External entities are represented by rectangles. Entities supplying data are known as sources and those that consume data are called sinks. Data are stored in a data store by a process in the system. Each component in a DFD is labelled with a descriptive name. Process names are further identified with a number.

The Data Flow Diagram shows the logical flow of a system and defines the boundaries of the system. For a candidate system, it describes the input (source), outputs (destination), database (files) and procedures (data flow), all in a format that meet the user’s requirements.

The main merit of DFD is that it can provide an overview of system requirements, what data a system would process, what transformations of data are done, what files are used, and where the results flow.

Rules for constructing a Data Flow Diagram

  1. Arrows should not cross each other.
  2. Squares, circles and files must bear names.
  3. Decomposed data flow squares and circles can have same time.
  4. Choose meaningful names for data flow.
  5. Draw all data flows around the outside of the diagram.

from fastapi import APIRouter, Depends, HTTPException, status
from app.routers.auth import get_current_admin_user
from app.db.database import db

router = APIRouter(prefix="/admin", tags=["admin"])

@router.get("/stats")
async def get_dashboard_stats(current_user: dict = Depends(get_current_admin_user)):
    try:
        total_users = await db.users.count_documents({})
        total_movies = await db.movies.count_documents({})
        total_reviews = await db.reviews.count_documents({})
        total_messages = await db.messages.count_documents({})

        return {
            "total_users": total_users,
            "total_movies": total_movies,
            "total_reviews": total_reviews,
            "total_messages": total_messages
        }
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))
